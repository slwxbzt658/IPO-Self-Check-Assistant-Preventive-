## 4. 第一层:表象数据预警

### 4.1 裁判标准(Blue)

🔵【裁判标准（Blue）】：
    - **一般法律红线：** 《首次公开发行股票注册管理办法》第十二条规定，发行人必须具备“独立持续经营能力”，严禁“严重依赖导致丧失独立性”。
    - **特定行业监管逻辑：** 鉴于军工航空领域特殊的宏观垄断格局（中航工业、中国航发主导），监管**不绝对否决**高客户集中度（视其为行业普遍性），但**极度警惕**因高集中度导致的“附庸化与代工化”。当单一客户占比超 50% 甚至突破 70% 时，将强制触发最高级别的穿透审查，核实企业是否具备独立面向市场的议价权与技术壁垒。
### 4.2 事实抓取(Green)

- 🟢**【数据事实】：** 报告期内，公司对第一大客户（中航工业 A01 单位）的销售收入占比分别为 59.71%（2020年）、68.25%（2021年）和 70.43%（2022年1-6月），呈现极端的单向集中趋势。
- 🟢**【自述辩护事实】：** 公司在 V1.0 招股书“重大事项提示”中自述声明：下游客户集中度较高符合航空制造业的行业特性，公司已与主要客户建立长期稳定的合作关系，不存在持续经营能力发生重大不利变化的风险。

🟠【推演】：

字段说明:

| 字段                           | 含义                 | 用途             |
| ---------------------------- | ------------------ | -------------- |
| `stress_scenarios`           | 多档降价情境下的利润影响       | 量化"价格→利润"的传导斜率 |
| `critical_cut_threshold_pct` | 触发 30% 红线所需的最小降价幅度 | 越小越危险          |
| `redline_breached`           | 是否突破监管红线           | 支柱一据此判定致命性     |
| `fatal_profit_loss_cny`      | 极限情境下的具体利润损失金额     | 提供可引用的数字证据     |
| `evidence_spans`             | 原文证据片段             | 满足溯源要求         |
| `verdict_for_pillar1`        | 给支柱一的定性结论          | 供支柱一直接合成最终判决   |

> Schema 的作用是定义契约——也就是"调用方需要拿到什么数据才能做下一步判断",而不是描述模型内部怎么算。
> 写 schema 的方法很简单,问自己 3 个问题:
1. 支柱一调用这个 API,是为了拿什么结论? → 这家公司利润是否会被合法腰斩
2. 要支撑这个结论,需要什么定量证据? → 压力测试场景 + 是否触发 30% 红线
3. 裕鸢案例的真实回传是什么? → 已经写在 6.3.2 里了
